## 2. 当前未完成项（按模块）

## 2.1 Master

未完成项：
- 选主与脑裂防护的混沌验证仍不充分：缺少网络分区、抖动与重复切换下的系统级验证报告。
- rebalance/route-repair 参数调优未形成标准化基线（不同规模下的触发频率、恢复时延、误触发率）。

验收标准：
- 连续多轮故障注入后，`/active-master` 的 `epoch` 单调递增且无双主。
- 在可控负载下形成参数建议表（最小间隔、触发阈值、恢复超时）并附压测数据。

## 2.2 RegionServer

未完成项：
- WAL 最终决议与副本追赶虽已实现主链路，但缺少极端网络故障下的稳定性报告。
- 迁移链路缺少大文件、长时中断与重试风暴场景的系统化压测结论。

验收标准：
- 在网络分区/高抖动场景中，副本决议最终收敛且无长期悬挂项。
- 迁移失败可重复回滚，恢复后可再次迁移成功，且不产生脏文件残留。
- 故障恢复后副本拓扑回到目标副本数，读写一致性满足设计约束。

## 2.3 Client

未完成项：
- 路由指标已支持命令行/JSON/HTTP 导出，但尚未接入统一监控系统。
- 缺少跨天/跨周趋势聚合与告警策略（如 MOVING 重试率、读降级命中率、重定向峰值）。

验收标准：
- 指标可被统一采集系统持续抓取并可视化。
- 至少提供一套长期趋势看板与异常告警阈值建议。

## 3. 推荐推进顺序

1. 先补故障验证：完成 Master 与 RegionServer 的混沌场景覆盖。
2. 再补可观测：统一接入监控与趋势聚合，保证问题可定位。
3. 最后补参数收口：在可观测基础上完成恢复参数固化与压测基线沉淀。
